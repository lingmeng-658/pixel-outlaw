# 一、本轮目标

进入 Town Road 后先进入不限时的安全准备阶段，玩家主动越过触发线后才开始第二关遭遇：

```txt
进入 Town Road
→ 显示区域标题、准备区与战斗入口提示
→ 玩家可以停留、观察或返回 Dusty Outskirts
→ 玩家主动向北越过触发线
→ 关闭南侧返回入口
→ 第二关遭遇战只启动一次
```

安全准备区将来用于购买道具和关前强化，本轮只建立战斗触发和入口状态结构，不实现商店。

# 二、明确设计要求

## 1. 使用清晰的战斗状态

增加语义明确的状态，例如：

```ts
private levelTwoCombatStarted = false
```

命名可以按当前风格微调，但必须与 `levelTwoCompleted` 和 `levelTwoEncounter` 的职责分开：

- `levelTwoCombatStarted`：本次 Town Road 进入是否已经主动触发战斗；
- `levelTwoCompleted`：第二关是否已经完成；
- `levelTwoEncounter`：隐藏批次调度器本身。

新增状态必须覆盖初始化、Restart、Continue、离区、完成和场景重建路径。

## 2. 进入 Town Road 时保持安全

`enterTownRoad()` 在第二关未完成时应：

- 重置第二关运行时；
- 将战斗状态设为未开始；
- 不立即调用 `levelTwoEncounter.start()`；
- 让玩家出生在南侧准备区；
- 显示清晰的横向触发标记；
- 显示 `MOVE NORTH TO ENGAGE` 或同等简洁提示；
- 保持南侧返回入口开放，玩家此时可以回到 Dusty Outskirts。

第二关已经完成时，不显示触发线，也不重新启动遭遇。

## 3. 越过触发线后启动战斗

玩家向北越过具名阈值后，应一次性：

- 清除战斗触发标记和准备提示；
- 将战斗状态设为已开始；
- 使用当前 Phaser 游戏时间调用 `levelTwoEncounter.start()`；
- 给出一次简洁反馈，例如 `AMBUSH STARTED`；
- 关闭南侧返回入口；
- 同一次进入过程中不能重复触发。

触发阈值放入 `LEVEL_TWO_CONFIG`，例如 `combatTriggerY`，不要在流程代码中散落裸数字。

## 4. 战斗开始后关闭返回入口

这是本轮必须完成的关卡边界：

- 战斗尚未触发时，南侧入口保持开放，玩家可以返回 Dusty Outskirts；
- 战斗一旦触发，当前这一轮战斗进行期间南侧入口必须立即关闭；
- 玩家之后即使退回南侧道路，也不能触发 `transitionToDustyOutskirts()`；
- 入口关闭必须同时体现在视觉和实际判定上，不能只隐藏提示但仍允许传送，也不能只禁用传送却留下误导性的开放标记；
- Restart Level 或未完成第二关的 Continue 回到 Town Road 时，应重新进入准备阶段并重新开放入口；
- 第二关完成后的区域返回行为保持现有完成流程，不在本轮额外设计新的关后出口。

不要在多个函数里重复散落“能否返回”的复杂条件。优先让 `checkTownRoadReturnExit()` 调用一个语义清晰的判断，或通过一个集中 helper 同步管理入口的视觉与逻辑状态，例如：

```ts
canReturnFromTownRoad(): boolean
```

或：

```ts
setTownRoadReturnOpen(open: boolean): void
```

具体形式按当前结构选择，但 helper 必须职责单一，不能借机建立泛化场景框架。

## 5. 未开战时不得推进第二关

`updateLevelTwo()` 或其上层调用在战斗未开始时必须停止：

- 隐藏批次推进；
- 敌人生成；
- 争夺道具计划；
- Shield 进度生成；
- 第二关完成判断。

准备阶段不是“敌人暂时还没生成”，而是第二关战斗系统尚未启动。

## 6. 触发物和入口对象统一清理

战斗触发线、提示文字和南侧入口的视觉对象应有明确、统一的管理方式。

必须覆盖：

- 进入 Town Road；
- 战斗触发；
- 返回 Dusty Outskirts；
- Restart Level；
- Continue；
- 第二关完成；
- 场景重建和清场。

不能留下悬空文字、重复触发线、旧入口标记或失效对象引用。

## 7. 战斗启动后不可逆

同一次 Town Road 进入中：

- 向北越线只启动一次；
- 玩家向南退回准备区不能暂停、停止或重新启动遭遇；
- 入口不会因为玩家回到南侧而重新开放；
- 只有 Restart、Continue 或重新进入本关，才重新回到未开战准备状态。

# 五、验收标准

1. 进入 Town Road 后可以无限停留，期间不生成敌人、不推进隐藏批次、不生成争夺道具。
2. 战斗未触发时，玩家可以从南侧入口正常返回 Dusty Outskirts。
3. 玩家主动向北越过触发线后，第一批敌人才开始生成。
4. 战斗触发时，南侧返回入口的视觉标记和实际传送判定同时关闭。
5. 战斗触发后，玩家退回南侧不能返回 Dusty Outskirts。
6. 战斗只启动一次；越线后返回南侧不会暂停、停止或重启遭遇。
7. Restart Level 和未完成第二关的 Continue 均回到 Town Road 安全准备阶段，并重新开放南侧入口。
8. 第二关完成后重新进入不会显示触发线或重复战斗。
9. 区域标题、暂停、清场、第一关出口和现有第二关玩法保持正常。
10. 不出现重复入口标记、悬空触发提示、失效引用或旧状态污染下一轮。
11. `npm run build` 通过。
