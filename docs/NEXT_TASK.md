# 一、本轮目标

解决用户首次试玩时最明显的两个问题：

1. 第二关初期特殊敌人出现过快、身份辨识度不足，混战时难以一眼判断追踪者、冲锋者和枪手。
2. 金币拾取物会永久留在战场，需要在比道具效果更长的时间后自动消失。

本轮需要做到：

- 保留第二关“持续增压 + 敌人组合配合”的核心玩法。
- 保留最终阶段由小怪卡位、冲锋者迫使变向、枪手惩罚路线的组合压力。
- 让前三批按顺序介绍特殊敌人。
- 让三类敌人主要依靠轮廓而不是临时颜色区分。
- 将身份、强化状态、当前动作拆成不同视觉层级。
- 小幅放缓生成节奏，但不全面削弱第二关。
- 让第一关和第二关掉落的金币都具备统一生命周期。

# 四、三类敌人的轮廓辨识度

## 1. 总体原则

玩家应在快速移动和大量敌人同屏时，不依赖文字、不依赖临时 tint，也能通过轮廓判断敌人身份。

身份视觉优先级：

```txt
基础纹理 / 轮廓 = 敌人身份
头顶图标或脚下标记 = 永久强化状态
预警线 / 枪口闪光 / 动作特效 = 当前行为
黄色 ! = 当前主要抢夺者
```

不要再让一个整身 tint 同时承担敌人身份、强化状态、命中反馈、蓄力和抢夺者等多种语义。

## 2. 普通追踪者

保持为视觉背景杂兵：

- 继续使用最小、最简单的轮廓。
- 保持暗红色主色。
- 不增加生命点。
- 不改变碰撞体、速度、生命和一枪击杀规则。

## 3. 冲锋者

重新绘制程序生成纹理，使轮廓明显不同于追踪者和枪手：

- 整体更宽、更厚重，建议视觉尺寸约 `38 × 32`，具体像素可按现有风格微调。
- 加入明显的宽肩、护肩、前倾尖角或箭头状胸部元素。
- 保持橙色 / 土黄色身份主色。
- 即使转为灰度截图，也应能靠“宽、重、向前突出”的形状识别。
- 不得因为纹理增大而扩大实际碰撞难度；保持碰撞体接近当前大小，必要时显式设置 body size。

## 4. 枪手

重新绘制程序生成纹理：

- 整体比冲锋者更窄、更高。
- 必须有明显横向宽帽檐。
- 身侧或手部增加可识别的枪管轮廓。
- 保持深蓝色身份主色。
- 即使转为灰度截图，也应能靠“宽帽檐 + 枪”的轮廓识别。
- 同样不要扩大实际碰撞体。

不引入外部图片资源，继续使用 `src/textures.ts` 的程序生成像素纹理。

# 九、金币自动消失

## 1. 生命周期

为第一关和第二关所有金币拾取物统一增加生命周期：

```txt
总存续时间：12,000ms
最后警告时间：2,500ms
```

将数值集中配置，优先放在 `TIMING` 或语义清晰的金币配置中，不要把 `12000` 和 `2500` 散落在多个回调里。

12 秒长于当前 Coffee、Ammo、Buckshot 和 Shield 等主要限时效果，玩家有充分时间拾取，但金币不能永久堆积在战场。

## 2. 消失反馈

- 金币生成后正常漂浮。
- 剩余约 2.5 秒时开始明显但不过度刺眼的闪烁或透明度变化。
- 12 秒到期时，如果金币仍然 active，则停止相关 tween 并销毁。
- 玩家提前拾取时，停止金币自己的 tween / 闪烁，并保证到期回调不会重复处理或报错。
- 暂停游戏时，金币生命周期应随 Phaser 游戏时间暂停，不按真实世界时间继续流逝。
- 区域切换、死亡、重开和清场时，现有 `clearCoinPickups()` 仍应立即清理金币及其附属 tween / timer 状态。
- 金币自然消失不增加 `coinCount`，也不影响“已经掉落过”的关卡掉落计数，避免同一奖励反复补掉。

# 十三、验收标准

## 构建与范围

- `npm run build` 通过。
- 没有新增依赖。
- 第一关玩法、数值和流程不变。
- 存档结构和 Continue 逻辑不变。
- 枪手和冲锋者战斗数值除第二关生成间隔外不变。

## 批次节奏

- 第一批只直接生成普通追踪者。
- 第二批首次直接生成冲锋者，不直接生成枪手。
- 第三批首次直接生成枪手。

## 视觉辨识

- 静止截图中，即使不看颜色，也能通过轮廓区分三类敌人。
- 冲锋者明显更宽、更厚重。
- 枪手有明显宽帽檐和枪械轮廓。

- 第一关和第二关金币均在 12 秒后自然消失。
- 最后约 2.5 秒有清楚的即将消失反馈。

- 暂停期间生命周期暂停。
